
# 安全策略

## 报告安全问题

如果您发现了安全漏洞或安全问题，请立即通过以下方式报告给我们：
- 发送邮件到: security@codelens.ai
- 创建安全漏洞报告 Issue（使用安全漏洞模板）

我们会在收到报告后的 24 小时内回复您。

## 安全措施

### 1. API 密钥保护

- 配置文件存储在用户主目录下的隐藏文件夹 `~/.codelens-ai/` 中
- 文件权限设置为 0o600（仅用户可读写）
- API 密钥在内存中加密存储
- 支持环境变量配置，避免硬编码

### 2. 代码分析安全性

- 只分析本地文件，不访问网络资源
- 使用 AST 解析，不执行代码
- 对代码进行沙箱处理
- 限制 API 调用的权限范围

### 3. 数据隐私

- 工具在本地运行，代码不发送到外部服务器（除非使用 AI 优化功能）
- 用户可以选择使用本地模型或云模型
- 不会收集或存储用户的代码或信息

### 4. 依赖安全

- 定期更新依赖包到最新版本
- 检查依赖包的安全漏洞
- 使用经过验证的依赖包

### 5. 安全更新政策

- 及时响应安全漏洞报告
- 发布安全补丁
- 更新依赖包到安全版本
- 提供安全建议

## 安全最佳实践

### 用户应该注意

1. 谨慎使用自动修复功能，建议先查看建议再手动应用
2. 定期更新工具到最新版本
3. 检查依赖包的安全状态
4. 遵循最小权限原则配置 API 密钥
5. 使用安全的网络连接

### 开发人员应该注意

1. 遵循安全编码规范
2. 对用户输入进行充分验证
3. 定期进行安全审计
4. 修复发现的安全问题
5. 提供安全的 API 和服务
